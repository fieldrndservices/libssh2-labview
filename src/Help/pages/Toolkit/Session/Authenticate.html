Authenticate VI

Owning Palette: Session Class

Requires: LIBSSH2

Password Authentication

Authenticate the session using a username and password. This is considered the least secure method of authentication.

Error In describes error conditions that occur before this node runs. This input provides standard error in functionality.
Session In is the Session object representing an initialized session in the underlying libssh2 library.
Username is the username for the remote SSH server.
Password is the password associated with the username for the remote SSH server.
Sesssion Out is the same Session object representing an initialized session in the underlying libssh2 library from the input.
Error Out contains error information. This provides standard error out functionality.

Public Key File Authentication

Authenticate the session using a public-private key pair saved in files on the local machine.

Error In describes error conditions that occur before this node runs. This input provides standard error in functionality.
Session In is the Session object representing an initialized session in the underlying libssh2 library.
Username is the username for the remote SSH server.
Keys is the public-private key pair to use for authentication of the username. This is relative to the local file system, not the remote SSH server.
Passphrase is the passphrase associated with the public-private key pair when the pair was generated. A passphrase is optional, so leave empty if one was not used during the public-private key generation. Note, not specifying a passphrase is considered less secure and frowned upon by security experts.
Sesssion Out is the same Session object representing an initialized session in the underlying libssh2 library from the input.
Error Out contains error information. This provides standard error out functionality.
